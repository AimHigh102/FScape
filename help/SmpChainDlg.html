Sample Chaining

One of the first modules in FScape, I don't know if it still works, I tried and just get Out-of-Memory errors, so beware. As far as I remember this was taking from the CSound manual or some CSound tutorial, where an algorithm would jump between a bunch of sound files whenever it found the same sample value in two of the input files. Here you can specify a tolerance for the comparision (allowed deviation), a minimum time span to be congruent, a minimum spacing between two jumps and a crossfade length for performing a jump. Seek-speed I can't remember. The priorties of the sounds specify the probability for a certain sound to get chosen.
