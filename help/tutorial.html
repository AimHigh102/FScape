fscape tutorial

Warning: this isn't really a tutorial, just a collection of hints.

Once you've started FScape, the socalled main window will open which is actually a totally boring window because it will only display the en-vogue parole. However it provides a menu bar! Originally FScape was meant as an add-on for SoundHack, specifically as a processor of the STFT files created from SoundHacks "Spectral Analysis". The 'F' stood for "Frequency Domain" or "Fourier Space". The original kernel is now contained in the "Spectral Patcher" module and is accompanied by many other algorithms in the fourier domain as well as in in the time domain.

So in a nutshell, FScape is a collection of sound-file-processing algorithms, each of which can be invoked from the "New Module" menu. Each module usually consists of one or more input file gadgets and one or more output file gadgets as well as a set a module specific parameters. For example, the Fourier Translation module - which will calculate the discrete Fourier transform of a sound file - will allow you to select the transform direction (forward or backward), if you want to use complex or real input and output, whether the file size should be truncated or expanded to a suitable length (a sample frame number which equals a power of two); further, what gain should be applied to the result. Once you've made your settings, you click on the Process button on the frame's bottom right and the module will start rendering until a) it's finished (progress bar turns green), b) an error occurs (progress bar turns red and a dialog appears) or c) you stop the process by clicking the Stop button. Generally, when you prefer to normalize sounds ('normalized' gain setting), data is first written to temporary file and finally written out to the target file; most modules indicate the normalizing process by colouring the progress bar yellow. This is a hint for you that portions of the final sound file might be already available for preview (the sound file's length information is updated regularly, so you can use the Change Gain module to make a preliminary copy of the file being rendered at the moment).

The majority of the modules has an (albeit outdated) online help accessible from the frame's options menu. I try to complete the help files ASAP. The file menu allows you to load and save the current parameters of a module. A special preset file is kept in a folder named 'FScape' inside your home directory (i.e. /Users/admin ). This file is named as the module and ends with '.pst' (for preset). This is a quick-access mode of loading and saving settings.

When launching FScape for the first time, consider opening the preferences pane to see if things are o.k. for your computer. The temp folder field should be left to the initial value or left empty in which case FScape will write temporary files to the default location (/tmp on un*x systems) or set it to your preferred folder. Note that when processing large files in a series of steps, the temp folder may grow quite seriously. Temp files will be destroyed when processing is stopped or the application is quit.

Numeric parameters are adjusted using a socalled ParamField gadget which consists of a dial knob and a textfield. Press the mouse and "dial" clockwise or anti-clockwise to adjust parameters without the keyboard. There's a strange patent-pending acceleration mechanism in the dial wheel, you'll find out soon. Alt+click will reset the parameter usually to zero. The unit of the parameter is labelled right to the textfield. Sometimes the unit is switchable, for example in the Resample module, the target rate can be expressed as

  • 'Hz' : absolute sampling rate
  • '+/- semi' : rate expressed in semitones relative to the original rate. Semitones are defined in the equal tempered scale for twelve semitones per octave. Decimals are used to express resampling in cents.
  • '+/- percent' : rate expressed as a percent offset to the original rate. E.g. if the source file has 44.1 kHz, choosing +25% will lower pitch by 3.9 semitones or if you choose to upsample your material (uncheck 'Keep old rate') create a file with 55125 Hz sampling rate.
  • '+/- Hz' : rate offset in Hertz relative to the original rate.

Relative units require a reference value which in this example is the source file's rate. Therefore, switching the unit will only transform the values correctly if you have set the input file before. Sometimes you'll see a special unit 'beats' which used to be a user-definable resolution on Mac OS 9. This is by default set to 120 BPM but the corresponding preference pane will reappear in one of the next versions, being useful to adjust offsets and length on a sample level resolution. Note that you have to confirm each textfield change by hitting return.

Files are specified using socalled PathField gadgets. These consists of a textfield with the Java-style (un*x) path to the file. On the right there's a small harddisk icon with an arrow. Clicking on this icon will bring up a file chooser. Dragging files from the finder onto this icon (on Mac OS X) will replace the PathField contents. You can store nine of your favourite folder names by hitting META-(=apple)-numpad-1...9 and recall them by hitting META+numpad-1...9. Output file names are usually derived from input file names. When you change the output file name, FScape tries to find out if this is a variant of the input file name. The algorithm is very simple and if it cannot deduce a rule for your output name, changing the input file will not alter the output file automatically anymore. To turn back to auto-mode, clear the textfield and hit return. So beware to not overwrite your files. FScape will not warn you if you are going to overwrite a file. However, a blueish colour in output file gadgets indicates that a file denoted by that name already exists. Note that you have to confirm each textfield change by hitting return.

FScape cannot playback sound files by itself, you will need to open the sound files in another application such as the QuickTime player. You can press the mouse over the output file's path button (the harddisk shaped icon) and drag the mouse onto the QuickTime player icon in the Finder's Dock to open that file quickly. For some reason, you cannot drag the path onto an application alias on the desktop.

Below the PathField gadgets there's often a file info field (for input files) or a file format selection pane (for output files). Recognized input sound files can be AIFF, .snd/.au, IRCAM and WAVE. Sample formats 8, 16, 24, 32 bit int and 32 and 64 bit float are recognized. Note that WAVE 8 bit is buggy but I guess you won't want to use it. Also note that FScape was written on a 'Big Endian' computer and that on Linux i386 and Windows you might have trouble with your .snd/.au and IRCAM files, this is not yet tested. AIFF and WAVE should work perfectly, though. These are also the output formats available. Note that FScape is working with 32bit floating point format internally, therefore it's not possible to write 64bit floating point (which would be merely redundant), also 8bit output is not supported. FScape can handle any number of channels, however some modules are designed to work with a specific number of channels.
